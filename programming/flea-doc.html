The Manual That Nobody Loves to Read

As you're aware, Flea serves as a static website generator. While its speed isn't confirmed, it boasts both minimalistic and user-friendly attributes. Functioning seamlessly, it extracts content from Markdown files situated within a designated folder following a distinct structure. Subsequently, the generated output finds its place in the "flea-site" directory, located within the flea folder. This design philosophy stems from my desire to prioritize writing over intricate configuration or extensive customization options. And that's the entirety of its functionality – succinct yet effective.

The source folder, serving as the content repository, should consist of at least the following three files or folders: an index.md serving as the main page's content; a config.json file for website configuration (don't worry, there are just five customizable parameters in total); and a folder named imgs intended to house all the images used within the website.

Feel free to toss in any other folders and Markdown files under the content directory (just avoid naming them "imgs" and "index.md"). Flea will take care of structuring your website automatically. It creates a global navigation bar based on the order of index.md, sub-folders, and other Markdown files. Here's an example of the typical structure:

.
├── index.md
├── config.json
├── imgs
│   └── flea.jpg
├── blog
│   ├── my-back-itches.md
│   └── go-see-a-doctor.md
└── about.md

config.json

Within the "config.json" file, you have the ability to customize five key parameters: "lang," "title," "author," "footer," and "nav." These parameters allow you to set the website's language, title, author details, footer content, and navigation bar respectively. The first four parameters are string inputs, while the "nav" parameter accepts an array of strings utilizing Markdown syntax. Here's a possible example:

{
  "lang": "en-US",
  "title": "My New Iron Site",
  "author": "Tony Stank",
  "footer": "Generated by <a href=\"https://github.com/undr22/flea\">a metal flea</a>",
  "nav": [
    "[Home](/)",
    "[Daily Life](/blog)",
    "[About My shiny clothes](/about.html)"
  ]
}

Folders

Nested folders aren't permitted within the content directory. To put it simply, you can only add one additional layer of folders within the content folder. Pages of inner folders are displayed as directories, showcasing the pages rendered from their internal Markdown files.

Nonetheless, you have the option to include an index.md within an inner folder. This allows you to provide details about the folder. Unlike other files, this index.md won't be visible in the directory list. Instead, its content is directly read and displayed before the directory listing.

Markdowns

Flea comprehensively supports all prevalent Markdown syntax, encompassing essentials like bold, italics, strikethrough, images (automatically centered), quotes, code blocks, lists, tables, footnotes, and more. Nonetheless, it doesn't natively handle direct HTML statements. You might find my Markdown cheatsheet helpful for quick reference.

In the beginning of a Markdown file, you can use YAML syntax to include three types of information: "title", "date", and "tags". These specific details are treated as metadata and are displayed in a distinct manner, contributing to the layout of your page. The syntax looks like this:

---
title: the Flea Manual
date: 2023-08-10
tags: [flea, doc, animal]
---

Note: All index.md files do not support metadata.

Others

The default 404 page will solely feature a primary "404" heading. If you wish to personalize the 404 page, you can create a 404.md within the content folder. This file does not support metadata either.
